# Task Review System

Two-tier review system for task files generated by the Generate Tasks skill. Tier 1 (local Ollama) runs on every generation; Tier 2 (Codex CLI) runs on escalation.

## Overview

The Task Review System catches routine task-generation defects before execution:
- Missing required sections (## Targeted Tests, ## Success Criteria, etc.)
- Anti-pattern violations (dict.get fallbacks, string concatenation for codegen, etc.)
- Missing test coverage
- Cross-task issues (dependency cycles, missing quality gates) — Tier 2 only

**Design goals:**
- Tier 1 effectively free per iteration (<30 seconds/task)
- Tier 2 provides second-opinion review from a different model family
- Machine-readable review contract (JSON) for automated fix application

## Architecture

```
Claude Code → Generate Tasks → task-NN-TT-{slug}.md

                  ↓ review.py --phase NN

        Tier 1: Local Ollama Reviewer
        qwen2.5-coder:14b (primary)
        deepseek-r1:32b (fallback)
        Per-task scope (32K context)

                  ↓ writes

        task-NN-TT-{slug}.review.local.json

                  ↓ (escalation logic)

        Tier 2: Codex CLI Reviewer (optional)
        GPT-5.x-codex
        Phase scope (200K context)
        Filters out Tier 1 findings

                  ↓ writes

        phase-NN.review.codex.json

                  ↓ /apply-reviews skill

        Apply fixes to task files
        Mark applied in .review.applied.json

                  ↓ manual --verify

        Re-run Tier 1 to confirm fixes cleared
```

## CLI Usage

### Review a single task

```bash
python scripts/library/review/review.py --task d:/datrix/datrix-common/.tasks/phase-43/task-43-01-foo.md
```

Output: `task-43-01-foo.review.local.json`

### Review an entire phase (Tier 1 only, default)

```bash
python scripts/library/review/review.py --phase 43
```

Output: `*.review.local.json` for each task in phase 43.

### Review with manual Tier 2 escalation

```bash
python scripts/library/review/review.py --phase 43 --codex
```

Runs Tier 1, then unconditionally runs Tier 2 (Codex CLI).

### Review with threshold escalation

```bash
python scripts/library/review/review.py --phase 43 --codex-on-threshold
```

Tier 2 runs if:
- Any blocking finding, OR
- Total major+blocking findings ≥ threshold (default 5)

### Review with phase-gate escalation (recommended)

```bash
python scripts/library/review/review.py --phase 43 --codex-phase-gate
```

Tier 1 runs per task. After all Tier 1 fixes applied and verified, Tier 2 runs as final quality gate.

### Re-review after applying fixes (manual verification)

```bash
python scripts/library/review/review.py --phase 43 --verify
```

Only re-reviews tasks with `.review.applied.json` markers. Writes `*.review.local.verified.json`.

## Configuration

Edit `scripts/review/config.toml`:

```toml
[tier1]
endpoint = "http://10.94.0.100:11434"
primary_model = "qwen2.5-coder:14b"
fallback_model = "deepseek-r1:32b"
context_window = 32768
fallback_after_failures = 2
target_latency_seconds = 30

[tier2]
enabled = true
mode = "phase-gate"          # off | manual | threshold | phase-gate
threshold_blocking = 1        # any blocking finding triggers Tier 2
threshold_total_major = 5     # or this many major+blocking combined
codex_command = "codex"
skill_name = "review-tasks"
auth_method = "plus"         # "plus" (codex login) or "api_key" (OPENAI_API_KEY env var)

[paths]
datrix_root = "d:\\datrix"
review_artifacts = "d:\\datrix\\.review"
canonical_modules_cache = "d:\\datrix\\.review\\.canonical-modules-cache.json"
```

## Workflows

### Fast iteration (local-only)

```bash
# Generate tasks
/generate-tasks design-doc.md

# Review locally (Tier 1 only)
python scripts/library/review/review.py --phase 43

# Apply fixes (in Claude Code)
"Apply reviews to phase 43"

# Verify fixes cleared
python scripts/library/review/review.py --phase 43 --verify

# Execute tasks
/execute-tasks --phase 43
```

### Pre-merge quality gate

```bash
# Review with Codex phase-gate
python scripts/library/review/review.py --phase 43 --codex-phase-gate

# Tier 1 runs per task
# Apply Tier 1 fixes: "Apply reviews to phase 43"
# Re-verify: python review.py --phase 43 --verify
# Tier 2 runs as final gate over cleaned phase

# If Tier 2 finds additional issues, apply and verify again
```

## Exit Codes

- `0` — All reviews pass (or only warnings)
- `1` — Blocking findings exist
- `2` — Parse failures (Tier 1 models failed to produce valid JSON)
- `3` — Infrastructure errors (Ollama unreachable in local-only mode, config missing)

## Review JSON Schema

All reviewers emit JSON conforming to this schema:

```json
{
  "schema_version": "1.0",
  "source": "local" | "codex",
  "model": "qwen2.5-coder:14b" | "deepseek-r1:32b" | "gpt-5.x-codex",
  "scope": "task" | "phase",
  "target": "task-NN-TT-slug.md" | "phase-NN",
  "generated_at": "ISO-8601 timestamp",
  "verdict": "pass" | "warnings_only" | "needs_fixes" | "blocking",
  "findings": [
    {
      "id": "F-001" | "C-001",
      "severity": "blocking" | "major" | "minor" | "nit",
      "category": "anti_pattern" | "missing_section" | "broken_reference" | "dependency_error" | "test_coverage" | "doc_coverage" | "ambiguity" | "scope" | "other",
      "location": "task file section or line reference",
      "description": "What is wrong",
      "evidence": "Quoted text from task file",
      "suggested_fix": "What to change",
      "rule_reference": "ai-agent-rules/prohibited-patterns.md §N" | null
    }
  ],
  "summary": "One-paragraph summary"
}
```

## Troubleshooting

### Ollama unreachable

**Symptom:** `ERROR: Ollama unreachable. Exiting.`

**Cause:** Ollama server not running at configured endpoint.

**Fix:**
1. Verify Ollama running: `curl http://10.94.0.100:11434/api/tags`
2. Check `config.toml` endpoint matches Ollama server
3. If using Codex flags (`--codex`, `--codex-phase-gate`), Tier 1 will be skipped automatically

### Parse failure stub reviews

**Symptom:** Review has `verdict: "warnings_only"` with finding `"Tier 1 models failed to produce valid JSON"`

**Cause:** qwen and deepseek both failed to emit parseable JSON.

**Fix:**
1. Check prompt length (may exceed model's effective context window)
2. Simplify task file or reduce canonical modules context
3. Retry with different model in config.toml

### Codex rate limited

**Symptom:** `Codex rate limit reached`

**Cause:** ChatGPT Plus daily quota exceeded.

**Fix:**
1. Wait for quota reset
2. Review Tier 1 artifacts (findings still available)
3. Apply Tier 1 fixes manually and re-run Tier 1 only

## Success Criteria (from Design)

A successful review system demonstrates:
1. Local review runs reflexively on every Generate Tasks invocation
2. Mean Tier 1 wall time <30 seconds/task (~4 minutes for 8-task phase)
3. Tier 1 catches >50% of issues that would surface during agent execution
4. Tier 2 surfaces ≥3 material issues across 3 phases that Tier 1 missed (evaluation gate)
5. Full pipeline (generate → tier 1 → apply → verify → tier 2 → apply) under 15 minutes for 8-task phase
6. Orchestrator codebase <300 lines Python (excluding prompts)
7. Canonical-modules cache hit rate >90% during normal development
