# Evaluation Workflow Guide

This guide explains how to use the two-tier evaluation system for Datrix generated code.

## Overview

The evaluation system is split into two skills:

1. **`/evaluate-generated`** (Quick Mode) -- Fast project-level evaluation + generates service prompts
2. **`/evaluate-generated-service`** (Deep Mode) -- Comprehensive single-service evaluation with semantic verification

## When to Use Each Skill

### Use `/evaluate-generated` when:
- Evaluating a large project (10+ services) like CurvAero
- You want a quick project-wide overview
- You need to identify which services have issues
- You want to generate evaluation prompts for selective deep-dive

### Use `/evaluate-generated-service` when:
- Evaluating a single service in detail
- You need semantic verification of transpiled code
- Running prompts generated by `/evaluate-generated`
- Debugging specific service generation issues

## Workflow for Large Projects (CurvAero)

### Step 1: Quick Project Evaluation

```
/evaluate-generated

SOURCE: D:\datrix\datrix-projects\curvaero\datrix-app\system.dtrx
GENERATED: D:\datrix\.generated\python\docker\curvaero
```

**What happens:**
- Creates timestamped directory: `D:\datrix\eval\2026-05-11-143022-curvaero\`
- Reads `system.dtrx` to identify all 36 services
- Reads manifests (no deep file scanning)
- Verifies project-level infrastructure (docker-compose, gateway, observability)
- Generates `project-evaluation-quick.md`
- Generates 36 service prompt files (one per service)

**Time:** ~3-5 minutes for CurvAero

### Step 2: Review Project Report

Open and review:
```
D:\datrix\eval\2026-05-11-143022-curvaero\project-evaluation-quick.md
```

This shows:
- Which services are missing
- Project-level blockers
- Infrastructure completeness
- List of generated service prompts

### Step 3: Deep-Dive on Specific Services

#### Option A: Evaluate One Service
```
/evaluate-generated-service

PROMPT_FILE: D:\datrix\eval\2026-05-11-143022-curvaero\service-aviation-data-service.prompt.md
```

**What happens:**
- Reads `aviation-data-service.dtrx` in full
- Reads all config YAMLs for this service
- Catalogs entities, blocks, APIs
- Scans generated service directory in detail
- Performs semantic verification (computed fields, hooks, validations, endpoints)
- Detects dead code (unused modules, dependencies)
- Generates `service-aviation-data-service-evaluation.md` in the same directory

**Time:** ~2-4 minutes per service

#### Option B: Evaluate Multiple Services in Parallel (Script)

Use `evaluate-services.ps1` to run all service prompts in parallel batches:

```powershell
cd D:\datrix\eval\2026-05-11-143022-curvaero

powershell -File "d:/datrix/datrix/scripts/dev/evaluate-services.ps1" `
    -SourceDir "d:\datrix\datrix-projects\curvaero\datrix-app" `
    -GeneratedDir "d:\datrix\.generated\python\docker\curvaero"
```

This runs 5 services concurrently (configurable via `-Parallel`). See `dev/quick-reference.md` for full parameters.

#### Option C: Evaluate Multiple Services Manually

Spawn multiple agents to evaluate services concurrently:

```
# Terminal 1
/evaluate-generated-service
PROMPT_FILE: D:\datrix\eval\...\service-aviation-data-service.prompt.md

# Terminal 2 (new session)
/evaluate-generated-service
PROMPT_FILE: D:\datrix\eval\...\service-airspace-service.prompt.md

# Terminal 3 (new session)
/evaluate-generated-service
PROMPT_FILE: D:\datrix\eval\...\service-obstacle-service.prompt.md

# ... etc
```

Or use the Task tool to spawn parallel agents programmatically.

#### Option D: Evaluate All Services Sequentially

```
# For each service, run:
/evaluate-generated-service
PROMPT_FILE: D:\datrix\eval\...\service-{service-name}.prompt.md
```

**Time:** ~2-4 minutes × 36 services = 72-144 minutes total

### Step 4: Review Service Reports

After service evaluations complete, review individual reports:
```
D:\datrix\eval\2026-05-11-143022-curvaero\service-aviation-data-service-evaluation.md
D:\datrix\eval\2026-05-11-143022-curvaero\service-airspace-service-evaluation.md
...
```

Each report shows:
- Service completeness (missing entities, endpoints, migrations)
- Semantic correctness issues (transpilation bugs)
- Dead code (unused modules, dependencies)
- Service deployment readiness
- Generator fixes needed

### Step 5: Fix Generator Issues

Service reports trace issues back to generator code. For example:

**Issue:** Missing lifecycle hook transpilation
**Generator:** `datrix-codegen-python/src/.../entity_generator.py`
**Fix:** Update lifecycle hook template

Make fixes in generator code, regenerate, and re-evaluate.

## Workflow for Small Projects (< 10 services)

For small projects, you can still use the quick mode to get organized, OR you can skip it and go straight to `/evaluate-generated-service` for each service.

### Option 1: Use Quick Mode (Recommended)
Same as large project workflow - get project overview + service prompts.

### Option 2: Direct Service Evaluation
```
/evaluate-generated-service
```

## Tips for Large Projects

1. **Start with quick mode** -- Always run `/evaluate-generated` first to get organized
2. **Focus on priority services** -- Don't evaluate all 36 services unless needed; pick the most critical ones
3. **Run in parallel** -- Use multiple Claude Code sessions or Task agents to evaluate services concurrently
4. **Aggregate findings** -- After service evaluations, create a summary of common generator issues
5. **Fix generators, not output** -- Service reports trace to generator code; fix there, then regenerate all

## Comparison: Old vs New Approach

### Old Approach (Single-Pass)
```
/evaluate-generated (full mode)
→ Reads all 36 .dtrx files
→ Reads all config YAMLs
→ Scans all generated files
→ Performs semantic verification on everything
→ FAILS with context overflow or timeout
```

**Problem:** Doesn't scale to CurvAero

### New Approach (Two-Tier)
```
/evaluate-generated (quick mode)
→ Reads system.dtrx only
→ Reads manifests only
→ Quick project checks
→ Generates 36 service prompts
→ ✅ Completes in 3-5 min

Then (selectively):
/evaluate-generated-service × N
→ Deep eval per service
→ ✅ Each completes in 2-4 min
```

**Benefit:** Scales to any project size

## Troubleshooting

### "Service directory not found"
- Verify service name in `system.dtrx` includes matches filename
- Check that generation completed successfully
- Look at manifest files to see if service was generated

### "Can't read manifest files"
- Ensure generation pipeline wrote manifests to `.datrix/manifests/`
- Check that paths in skill invocation are correct

### "Prompt file not found"
- Make sure you ran `/evaluate-generated` first
- Check that evaluation directory was created
- Verify prompt file path matches what was reported

### "Too many context tokens"
- This should NOT happen with the new two-tier system
- If it does, the service is extremely large (1000+ entities?)
- Consider splitting the service or contact support
